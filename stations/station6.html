Why Vibe Coding Doesn't Cut It - Being the Human in the Loop

Prof. Teeters calls it "vibe coding" - just throwing prompts at AI and hoping something good comes out. I used to do this! "Make it look professional" *paste*. "Change it to be more modern" *paste*. No understanding, no system, just vibes. It was like cooking by throwing random ingredients in a pot and hoping for dinner!

The Hard Truth Prof. Teeters Taught Me: You, the developer, are the one who needs to be the director of the operations. Your thinking is more expansive and more flexible, and your ability to hold memory and context in your mind to pull it up as needed far exceeds your AI Assistant.

This is why you need to study and understand the development and design concepts that you and your AI Collaborator will then implement. Use your AI Assistant for what it does best.

Learning to Prompt

My first AI color prompts were embarrassing. "Give me good colors" got me a rainbow. "Make it pop" got neon everything. Prof. Teeters taught me the formula: context + mood + constraints. "E-commerce site for outdoor gear, earthy and trustworthy, need 5 colors including success/error states, must meet WCAG 2.2 Level AA contrast" - NOW we're talking!

The Magic Formula Prof. Teeters Taught Me

  1. Context: What's this for? (website, app, presentation)
  2. Mood: What feeling? (professional, playful, serious)
  3. Constraints: What rules? (WCAG 2.2 Level AA, brand colors to match, number of colors)
  4. Format: Request HSL values! (builds on Station 1 learning)
  5. Terminology: Use "CSS custom properties" not "CSS variables"!
HAP's Old Way (Vague Prompts):
"nice website colors"
"make it modern"
"something that pops"
What I Learned (Specific Prompts with Correct Terms):

- Professional but approachable mood

- Need: primary, secondary, success, warning, error
- All must meet WCAG 2.2 Level AA contrast (4.5:1) on white backgrounds
- Include hover state variations using lighter/darker L values
- Provide as CSS custom properties (not variables)
- Use :root selector with proper custom property syntax"

Teaching AI Our Standards

The breakthrough was training AI to follow our Stations 1-5 standards. I explicitly tell it what we learned!

My System Prompt for AI:
Help me create a color system following these requirements:
1. Use HSL format only (no hex codes)
2. Create CSS custom properties (not "variables")
3. Use semantic naming (--primary, not --blue)
4. Follow the 60-30-10 distribution rule
5. Ensure WCAG 2.2 Level AA conformance (4.5:1 minimum)
6. Include systematic scales (100-900 for grays)
7. Apply color wheel harmony (complementary/analogous)
8. Use modern features like color-mix() where appropriate

My AI Workflow for Color Systems

  1. Request a base system: "Create CSS custom properties for a color system using HSL"
  2. Ask for scales: "Generate a gray scale from 100-900 using consistent lightness steps"
  3. Request semantic names: "Use semantic naming (primary, secondary) not color names (blue, green)"
  4. Get variations: "Create hover states using color-mix() like Station 5"
  5. Verify everything: Check contrast ratios yourself!
Advanced Technique: Context Files

If you're using an AI Agent that allows for markdown files for context (and they all do now), write your rules in those files and alert your AI Agent that those rules are to be followed. For example, create a color-standards.md file:

# Project Color Standards
1. Always use HSL format, never hex codes
2. Use CSS custom properties, NOT "CSS variables"
3. Semantic naming only (--primary, never --blue)
4. All colors must meet WCAG 2.2 Level AA (4.5:1 ratio)
5. Follow 60-30-10 distribution rule
6. Provide scales using 100-900 numbering

Then tell your AI: "Follow the rules in color-standards.md for all color work."

‚ö†Ô∏è Warning About Drift

Common AI Mistakes to Fix

Prof. Teeters taught me these corrections:

AI Says ‚Üí You Correct

  • "CSS variables" ‚Üí "CSS custom properties"
  • "Here's a variable" ‚Üí "Here's a custom property"
  • $primary (Sass) ‚Üí --primary (CSS)
  • "Store in variables" ‚Üí "Define as custom properties"
Real Example from My Chat:
AI: "I'll create CSS variables for your color scheme"

ME: "Please use the correct term 'CSS custom properties' -
     variables is incorrect. Also provide HSL values, not hex."

AI: "You're right! Here are CSS custom properties using HSL:"

Code Examples: Before and After

‚ùå Bad AI Result (Multiple Mistakes)

/* AI Response to "nice website colors" */
/* AI says: "Here are your CSS variables:" - WRONG! */
:root {
  --blue: #3366CC;      /* Hex AND color name! */
  --green: #00FF00;     /* Pure green - too bright */
  --red: #FF0000;       /* No system, just colors */
}
‚úÖ Corrected Result (After Teaching AI):
/* Corrected: CSS custom properties with HSL */
:root {
  /* Primary color system (custom properties, not variables!) */
  --primary: hsl(210, 50%, 50%);
  --primary-dark: hsl(210, 50%, 40%);
  --primary-light: hsl(210, 50%, 60%);

  /* Semantic custom properties for states */
  --success: hsl(134, 61%, 41%);  /* Meets WCAG 2.2 Level AA */
  --warning: hsl(45, 70%, 35%);   /* Dark enough for contrast */
  --error: hsl(0, 65%, 45%);      /* Accessible on white */

  /* Gray scale using custom properties */
  --gray-100: hsl(0, 0%, 95%);
  --gray-500: hsl(0, 0%, 50%);
  --gray-900: hsl(0, 0%, 10%);

  /* Neutrals for true white and black */
  --white: hsl(0, 0%, 100%);
  --black: hsl(0, 0%, 0%);
}

See the difference? Semantic naming, HSL values, correct terminology, and accessible contrast ratios!

ü§ñ HAP's Quick Reference Tips

1

Correct Terminology Matters

Always say "CSS custom properties" not "variables." This trains AI to use correct terms and shows you understand the specification. Prof. Teeters says precision in language leads to precision in code!

2

Be Explicit About Format

Don't just ask for "colors" - specify "HSL custom properties in the :root selector." The more specific, the better the output. Remember: hex codes are for copying, HSL is for understanding!

3

Teach AI Your Standards

Reference what you learned: "Like Station 2 taught us, use semantic naming for custom properties." AI will adapt to your requirements. Create a standards file if your AI supports context files!

4

Fix the Training Data Problem

AI learned from tutorials that say "CSS variables" everywhere. You have to actively correct this. Think of it as being the human in the loop, fixing the training data mistakes!

5

Iterate, Don't Accept

First suggestion rarely perfect. Ask for variations, adjustments, alternatives. It's faster to refine than to start over. Think of AI as a brainstorming partner.

6

Keep What Works, Fix What Doesn't

Like 3 colors but not the 4th? Keep those three and ask for alternatives for just that one. You don't have to take all or nothing.

HAP's Final Thought

Prof. Teeters taught me that AI mirrors the internet's mistakes - like saying "CSS variables" instead of "custom properties," or defaulting to hex instead of HSL. As developers, we're the quality control! We correct the terminology, enforce the standards, and verify the claims. AI speeds up our work, but WE ensure it's correct.

Remember: custom properties, not variables; HSL, not hex; semantic names, not color names. Be the human in the loop who insists on doing it right!
